context_state 字段分层

当前草案把字段分为核心字段、偏好字段、元字段。核心字段决定是否能推荐。

层级字段说明
核心字段activity_goal departure_area time_window transport_hint ability_hint推荐充分条件主要依赖这些。
偏好字段preference_tags avoid_tags terrain_tolerance scenery_preferences safety_priority supply_requirement影响选择卡、召回加权、扣分和解释。
元字段confirmed_fields missing_fields field_sources用于可信度、展示、调试和恢复。

sufficiency_check 推荐充分条件

这里决定系统什么时候停止追问、开始生成候选线路。

stateDiagram-v2 [*] --> Collecting Collecting --> WaitingUser: 缺少核心字段或冲突 WaitingUser --> Collecting: choice_result 写入 context_state Collecting --> Recommendable: 满足推荐充分条件 Recommendable --> Recommended: route_retrieval + candidate_routes Recommended --> FollowUp: 用户追问 FollowUp --> Collecting: 用户修改关键条件 FollowUp --> Recommended: 解释现有推荐

请选择本轮充分条件

需要权衡

  • 问太少：推荐可能偏。
  • 问太多：用户觉得慢。
  • 场景化：实现略复杂，但户外更稳。

第一轮选择卡问什么

一轮最多 3 个问题。第一轮不追求问完，只补足最关键的不确定性。

用户输入建议第一轮选择卡
周末想出去走走出发地、出行类型、时间窗口
成都周末想看雪，但别太危险交通、强度、冰雪/路况接受程度
想带孩子轻松徒步出发地、时间、交通；安全默认高优先级
