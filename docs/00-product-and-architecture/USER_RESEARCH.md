# 用户洞察与问题定义

Status: active
Owner: project maintainer
Last reviewed: 2026-05-08
Source of truth: migrated from PRD module one.

## 目标用户

本项目面向轻中度户外用户。典型用户包括：

```text
大学生群体
大众徒步参与者
周末短途出行用户
已有基础户外意识但缺少系统规划能力的用户
```

他们不是完全小白，也不是极限探险者。他们通常知道轨迹、地图、天气等工具，但没有足够经验把多源信息合并成稳定判断。

## 用户痛点

### 1. 不知道去哪

信息很多，但难以决策。

```text
小红书：风景和种草强，但不能证明线路可走。
两步路：轨迹专业，但缺少近期天气、交通、路况和体验信息。
地图/天气/交通：各自有用，但无法形成完整线路决策。
```

### 2. 不知道能不能走

用户很难把专业轨迹指标映射到自己的能力：

```text
距离
爬升
海拔
坡度
路况
撤退点
交通可达性
```

### 3. 不知道是否可成行

户外出行不是只看线路本身，还依赖：

```text
天气
交通
返程
近期路况
封路/封山/积雪
补给和通信
```

这些信息分散在多个工具中。

### 4. 决策协作难

用户经常需要把规划发给朋友、群聊或自己后续回看。纯聊天记录信息密度不稳定，传统攻略又太重。

## 产品机会

市场不缺内容，也不缺轨迹工具。缺的是：

```text
从模糊需求到真实轨迹的桥梁
从内容种草到可执行判断的桥梁
从个人能力到线路强度的桥梁
从碎片信息到可分享方案卡的桥梁
```
