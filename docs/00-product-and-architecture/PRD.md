## 产品定位

智行户外是一款任务驱动的智能户外规划 Agent，面向轻中度户外用户，提供从“模糊需求”到“真实轨迹”的可决策线路方案。

核心价值：

```text
自然语言表达需求
结合真实线路资产与轨迹指标
结合用户能力画像
验证天气、交通、外部路况证据
生成可保存、可回看、可分享的方案卡
```

## 目标用户

目标用户是轻中度户外用户，例如大学生、大众徒步爱好者、日常出游人群。

用户特征：

```text
有出行意愿
具备基础风险意识
知道轨迹和天气等工具
不是完全新手，也不是极限探险者
缺乏稳定的信息整合和专业线路判断能力
```

## 核心痛点

```text
不知道去哪
内容很多，但推荐不等于适合自己。小红书种草强，但专业性不足；两步路轨迹强，但缺少风景、交通、天气和近期信息。

不知道能不能走
距离、爬升、海拔等指标难以和个人体能匹配。

不知道是否可成行
天气、交通、返程、近期路况分散在多个 App 和网页中。

决策协作难
约朋友时缺少信息密度适中、可解释、可转发的方案载体。
```

## 核心功能

### 自然语言对话规划与候选生成

用户输入模糊需求，例如“周末想看雪山但别太累”。Agent 抽取出发地、时间窗口、交通方式、同行者、能力状态和偏好目标，从线路资产库召回候选，输出 3 张极简候选卡片。

### 详情方案卡与多源证据验证

用户点击候选卡片后，系统展示距离、爬升、预计用时、适合原因、风险、预算、装备建议和证据来源。

### 导入历史轨迹生成能力画像

用户导入已完成轨迹，系统解析距离、爬升等指标，生成长期能力画像，用于推荐时的人线匹配。

### 非线性规划沉淀与管理

用户可把某个候选详情保存到“我的规划”，之后回看、分享或回到原 TripPlan 继续追问。

## 产品入口

MVP 移动端 H5 包含四个底部 Tab：

```text
出去走走
我的规划
线路
个人中心
```

## 差异化

```text
结构化轨迹资产 + 多源证据验证
基于真实完成轨迹的人线匹配
垂直户外 Agent，而不是通用聊天机器人
保存和分享导向的规划工作区，而不是一次性问答
```

## MVP 边界

PRD 中提到 PostgreSQL、PostGIS、pgvector、JSONB 等长期技术方向。当前 MVP 以迭代文档为准，可以先使用 SQLite/JSON 和轻量召回，不引入 PostGIS 或向量库，除非 ADR 明确升级。
